## 第1页：项目概述

### 一句话定位

**OpenOctopus** —— 全球首个 Realm-native 人生治理 Agent 系统。

不是另一个聊天机器人，而是将你的人生分域治理（宠物、父母、财务、工作...），并通过 **Summon（召唤）** 将任何现实对象转化为有记忆、有性格、可主动行动的 AI Agent。

### 核心使命

> 让每个人都拥有一个真正懂你的 AI 人生伙伴。

### 解决的三大痛点

| 痛点 | 现状 | OpenOctopus 方案 |
|------|------|------------------|
| **AI 工具碎片化** | ChatGPT、Notion、记账软件来回切换 | Realm Matrix 一站式管理所有人生领域 |
| **情感连接缺失** | 通用 AI 冷冰冰，无法建立长期关系 | Summon 机制赋予实体记忆与个性，建立情感纽带 |
| **隐私焦虑** | 云端 AI 存储敏感个人数据 | Local-first 架构，数据本地存储，可选云同步 |

## 第2页：核心创新 —— Summon 机制

### 什么是 Summon？

Summon 是 OpenOctopus 的核心差异化功能：将**任何现实实体**转化为**有生命的 AI Agent**。

```
原始数据 → 结构化实体 → SUMMON → 有记忆、有个性、可主动行动的 Agent
```

### 实体类型矩阵

| 实体类型 | 示例 | 召唤后能力 | 应用场景 |
|---------|------|-----------|----------|
| **Living（生命体）** | 宠物、家人、朋友 | 模拟对话、情感表达、记忆历史、主动关怀 | 宠物健康顾问、父母照护助手 |
| **Asset（资产）** | 汽车、房产、投资组合 | 状态监控、维护提醒、成本分析、价值追踪 | 车辆保养管家、投资分析师 |
| **Organization（组织）** | 医院、学校、公司 | 流程向导、联系人管理、预约协调 | 就医流程助手、学校事务管家 |
| **Abstract（抽象）** | 目标、项目、习惯 | 进度追踪、偏差预警、复盘建议、里程碑提醒 | 减肥教练、项目里程碑监控 |

### Summon 实战示例

```
用户："我下周要出差5天。"

Momo（召唤的宠物）："谁要喂我、遛我5天啊？！"
Mom（召唤的母亲）："妈说她正好想来看看——她可以照顾 Momo。"
Car（召唤的车辆）："出发前充满电？还是打车去机场更划算？"
Budget（财务 Realm）："预估差旅费 ¥3,500，记得保存发票。"

→ 系统自动生成《出差准备清单》等待用户确认
```

### 与 OpenClaw SOUL.md 的本质区别

| 维度 | OpenClaw SOUL.md | OpenOctopus Summon |
|------|------------------|-------------------|
| **性质** | 静态配置文件 | 动态实体转化过程 |
| **触发方式** | 手动编写配置 | 一键召唤，自动提取实体属性 |
| **记忆系统** | MEMORY.md 文件存储 | JSONL 时间序列 + 知识图谱关联 |
| **跨域协同** | 依赖 Gateway 路由 | Realm 间实体自然关联 |
| **用户门槛** | 需理解配置语法 | 零门槛，对话式召唤 |

| 维度 | OpenClaw | OpenOctopus |

| **跨域协调** | Gateway 路由转发 | 知识图谱实体关联 |

## 第4页：技术架构与实现

### 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **Runtime** | Node.js ≥22 + TypeScript 5.7+ | 严格类型安全 |
| **Gateway** | Express 5 + WebSocket (ws) | 双端口：WS 19789 (RPC) + HTTP 19790 (REST) |
| **数据库** | SQLite (better-sqlite3) | Local-first，可选 PostgreSQL/Supabase 云同步 |
| **向量检索** | pgvector / 本地向量库 | 按 Realm 分片存储 |
| **配置系统** | JSON5 + Zod 验证 | `~/.openoctopus/config.json5` |
| **包管理** | pnpm workspace | 8 个包组成的 monorepo |
| **测试** | Vitest | 51+ 单元测试 |
| **代码规范** | oxlint + oxfmt | 高性能 lint 和格式化 |

### Monorepo 结构

```
packages/
├── shared/     # 类型、错误、ID、日志、配置、RPC 协议
├── storage/    # SQLite、迁移、JSONL 会话、仓库
├── core/       # Realm 管理器、实体管理器、Agent 运行器、LLM 提供者
├── summon/     # SOUL.md 解析器、提示词编译器、召唤引擎
├── channels/   # 渠道适配器（Telegram、Discord、Slack、微信）
├── ink/        # Express+WS 网关、RPC 处理器、API 端点
├── tentacle/   # CLI 工具（tentacle 命令）
├── realmhub/   # 包注册客户端（Phase 3）
└── dashboard/  # Next.js Web UI（Phase 2）
```

### 与 OpenClaw 技术对比

| 技术点 | OpenClaw | OpenOctopus |
|--------|----------|-------------|
| **代码规模** | 430,000+ 行 TypeScript | ~50,000 行（Phase 1），精简聚焦 |
| **并发模型** | Lane Queue（串行默认）| 标准 async/await + 显式并发控制 |
| **Gateway** | 自研类型化 RPC | Express 5 + JSON-RPC |
| **配置** | 5+ 配置文件（SOUL/USER/IDENTITY/MEMORY/AGENTS）| JSON5 主配置 + REALM.md + SOUL.md |
| **渠道** | 12+ 渠道（更成熟）| 核心渠道（开发中） |
| **冷启动** | ZeroClaw 版本 10ms | 标准 Node.js 启动时间 |
| **存储** | SQLite + 嵌入模型 | SQLite + JSONL + 可选向量存储 |

### 本地优先（Local-first）架构

```
┌─────────────────────────────────────────────┐
│           OpenOctopus Core                  │
│  ├─ Realm Matrix（本地 SQLite）              │
│  ├─ Entity Memory（JSONL 时序）              │
│  └─ Agent State（本地状态机）                │
└──────────────────┬──────────────────────────┘
                   │ 可选云同步（加密）
                   ▼
        ┌──────────────────────┐
        │  PostgreSQL/Supabase  │
        │  （端到端加密）        │
        └──────────────────────┘
```

### 用户画像

**核心用户（极客/技术爱好者）**
- 25-40岁，开发者/产品经理/研究员
- 使用多个 AI 工具，感到碎片化困扰
- 关注隐私，偏好本地优先方案
- 愿意为个性化体验付费

**扩展用户（生活管理者）**
- 30-50岁，多角色身份（父母+子女+职场人）
- 需要管理复杂生活：宠物、父母、投资、健康
- 重视效率与情感连接
- 非技术背景，需要低门槛产品

### 竞品深度对比

#### 直接竞品分析

| 产品 | 类型 | 与 OpenOctopus 对比 | 我们的优势 |
|------|------|-------------------|-----------|
| **OpenClaw** | 开源个人 AI 助手 | 技术架构相似，但 OpenClaw 是任务导向，OpenOctopus 是人生治理导向 | Realm-native 组织 vs Workspace；Summon 动态实体 vs 静态配置；更低用户门槛 |
| **Notion AI** | 工作空间 + AI | Notion 是文档为中心，无领域划分，无实体召唤 | Realm Matrix 分域管理；Summon 情感化实体 |
| **Sintra AI** | 多 Agent 商业助手 | 最接近架构（中央大脑+专业 Agent），但仅面向商业 | 面向个人生活；Summon 实体化；RealmHub 分享 |
| **Uare.ai** | 个人数字孪生 | 专注自我克隆，无多领域管理 | 可召唤外部实体（宠物、家人）；跨域协同 |
| **Coze/扣子** | Agent 开发平台 | 无领域组织，无本地优先 | Realm-native；Local-first；开源 |

| 能力 | OpenOctopus | OpenClaw | Sintra | Notion AI | Uare.ai |

| 本地优先 | **✅** | ✅ | ❌ | ❌ | ❌ |
| 开源 | **✅** | ✅ | ❌ | ❌ | ❌ |
| 目标用户 | **个人生活** | 技术极客 | 企业 | 通用 | 个人 |

### OpenClaw 深度对比总结

| 维度 | OpenClaw | OpenOctopus | 策略含义 |
|------|----------|-------------|---------|
| **定位** | 个人 AI 助手/自动化工具 | 人生治理系统 | 避开直接竞争，差异化定位 |
| **核心抽象** | Workspace + Skill | Realm + Entity + Summon | 更贴近用户生活直觉 |
| **用户门槛** | 高（需写配置）| 低（对话式召唤）| 扩大目标用户群 |
| **情感连接** | 弱（工具属性）| 强（Summon 人格化）| 建立用户粘性 |
| **生态规模** | 60K+ Stars，30+衍生项目 | 新项目 | 利用 OpenClaw 教育的市场 |
| **渠道支持** | 12+ 渠道（成熟）| 核心渠道（开发中）| 快速跟进渠道支持 |
| **技术成熟度** | 43万行代码，生产级 | Phase 1，垂直切片 | 聚焦核心差异化功能 |

**关键洞察**：OpenClaw 证明了"本地优先 + 开源 + Agent"的市场需求，但其技术门槛限制了用户群。OpenOctopus 通过**Realm-native 架构**和**Summon 机制**降低门槛，面向更广泛的生活管理者群体。

| **隐私合规** | 高 | 中 | GDPR/CCPA 合规设计，Local-first 降低风险 |
| **开源商业化** | 中 | 中 | 云增值服务 + RealmHub 商店，保持代码开源 |
| **技术债务** | 中 | 低 | 严格测试（51+单元测试）、类型安全（TS严格模式） |
| **用户获取** | 中 | 中 | 借力 OpenClaw 教育的市场；差异化内容营销 |
| **Dot/Pi 前车之鉴** | 中 | 低 | 强调实用价值（任务执行）而非纯情感陪伴 |

### OpenClaw 生态依赖与应对

**机会**：OpenClaw 已教育市场"本地优先 + Agent"概念，降低用户教育成本。

**威胁**：OpenClaw 生态庞大，可能覆盖 OpenOctopus 的功能。

**应对策略**：
1. **差异化定位**：OpenClaw = 任务自动化工具，OpenOctopus = 人生治理系统
2. **生态兼容**：考虑支持 OpenClaw Skill 导入
3. **用户群体**：OpenClaw 面向极客，OpenOctopus 面向生活管理者
4. **技术借鉴**：学习 OpenClaw Gateway 设计，优化自身架构

## 第11页：愿景与差异化总结

### 长期愿景

> **"让 AI 成为你人生的治理者，而非只是任务的执行者。"**

OpenOctopus 不仅是一个工具，而是一种新的生活方式——你的宠物、家人、资产都以 AI 形态陪伴在你身边，形成有温度、有记忆、可成长的智能生态。

### 核心差异化（vs OpenClaw）

| 维度 | OpenClaw | OpenOctopus | 用户价值 |
|------|----------|-------------|---------|
| **定位** | AI 助手/自动化工具 | 人生治理系统 | 从"帮我做事"到"懂我的生活" |
| **组织方式** | Workspace | Realm Matrix | 按人生领域自然分类 |
| **实体化** | SOUL.md 配置 | Summon 一键召唤 | 零门槛创造有生命的 Agent |
| **记忆** | 文件存储 | 时序 + 知识图谱 | 跨领域关联记忆 |
| **目标用户** | 技术极客 | 生活管理者 | 扩大 10 倍潜在用户群 |

### 为什么是现在？

1. **LLM 能力成熟** — GPT-4/Claude 3 已具备复杂推理与长期上下文
2. **OpenClaw 验证市场** — 本地优先 Agent 需求已被证明
3. **隐私意识觉醒** — 用户对云端 AI 的担忧日益增长
4. **多 Agent 架构成熟** — CrewAI、AutoGPT 验证了 Multi-Agent 可行性
5. **数字孪生投资热** — Simile $100M、Viven $35M 证明市场潜力
