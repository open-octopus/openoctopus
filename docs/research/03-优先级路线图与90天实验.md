# 优先级路线图与 90 天实验

## 1. 先打哪三仗

### P1. 一人公司 AI Native 经营驾驶舱（最高优先）
- 定义：把“目标管理、内容生产、线索跟进、复盘”做成持续运行的代理系统。
- 为什么匹配：已有完整的一人公司规划文档与多项目执行经验。
- 不是插件式 AI：不是给 Notion 加总结按钮，而是让系统每天主动推进经营目标。

### P2. 脉冲升级版：关系与生活编排 OS
- 定义：在 `maichong` 上加入“多代理协商”（你、伴侣/家人、预算代理、时间代理、偏好代理）。
- 为什么匹配：已完成时间线 + 协作 + AI 助手思路，离 AI Native 只差“持续代理闭环”。
- 不是插件式 AI：不再是手动填日程，而是系统持续理解隐性意图并生成行动提案。

### P3. 多角色决策模拟器（argue-agent 产品化）
- 定义：针对“会议决策/项目路线/商业决策”做多角色代理辩论与反证。
- 为什么匹配：已有实时论点提取、事实核查、反驳建议能力。
- 不是插件式 AI：不是聊天问“这个方案行不行”，而是并行生成多立场推演和行动建议。

## 2. 90 天推进节奏

### 阶段一（Day 1-14）：统一技术底座
- 目标：打通协议与执行基础能力。
- 必做：
  1. 定义统一任务模型：`goal -> tasks -> agents -> evidence -> decision -> action`。
  2. 接入 MCP（检索/文件/外部工具）和至少一个多 agent 编排框架。
  3. 加入最小治理层：预算上限、人工审批点、失败回滚策略。
- 验收指标：
  1. 单个目标可被自动拆成 5-10 个任务。
  2. 每次任务有日志、证据链接、可重放。

### 阶段二（Day 15-45）：做出 3 个可演示场景
- 目标：每个优先方向做出可复用的“任务闭环 Demo”。
- 场景包：
  1. 经营驾驶舱：每周自动生成运营计划并跟踪执行偏差。
  2. 脉冲编排 OS：输入“想提升关系质量”，系统自动编排可执行计划并协同他人。
  3. 决策模拟器：输入一个策略，输出多立场辩论、风险清单、建议动作。
- 验收指标：
  1. 3 个场景都能连续运行 7 天。
  2. 每天最少触发 1 次“系统主动建议 -> 用户确认 -> 执行”。

### 阶段三（Day 46-90）：小规模真实用户验证
- 目标：验证“有人愿意持续使用并反馈价值”。
- 必做：
  1. 每个场景招募 5-10 个种子用户（朋友/同事/社群）。
  2. 每周一次定量复盘：任务完成率、建议采纳率、节省时间、留存率。
  3. 选择一个场景进入收费前测试（可先做预约和意向付费）。
- 验收指标：
  1. 周留存 > 40%。
  2. 核心动作（建议采纳）> 30%。
  3. 出现至少 3 个明确“愿意付费”反馈。

## 3. 可直接复用的资产清单
1. `maichong`：生活节律与协作交互底座。
2. `argue-agent`：实时论点提取、事实核查、决策对抗推演。
3. `english-agent`：学习闭环、行为追踪、通知与复盘机制。
4. `3d-agent` + `vedio-agent`：多模态内容生产与工具选型体系。
5. `brag/AI个人IP` 的规划文档：可直接作为经营驾驶舱的目标与 KPI 输入。

## 4. 三条产品化原则（避免再做成 Demo）
1. 先做“持续运行闭环”，再做“炫技能力”。
2. 每个 agent 行动必须有“可验证证据链”。
3. 没有治理的 agent 不上生产：预算、权限、审计缺一不可。

## 5. 建议下一步最小动作（本周）
1. 从 P1 起步：先做“每周经营计划自动生成 + 每日执行追踪 + 周报复盘”一个闭环。
2. 把 `maichong` 接入同一任务模型，验证生活场景是否可复用同一 agent 架构。
3. 用 `argue-agent` 补上“反证代理”，让系统具备自我纠偏能力。
